## Datasets

### Full Dataset (full_dataset_14_descriptors.csv)

**Description:** Complete training dataset with all molecules and 14 molecular descriptors.

**Size:**
- Rows: ~400-500 molecules
- Columns: 19 total (5 metadata + 14 descriptors + 1 duplicate target)
- Memory: ~100-150 KB

**Columns:**

#### Metadata Columns
| Column | Type | Description |
|--------|------|-------------|
| Unnamed: 0 | int | Index column (from pandas) |
| Atoms | str | Compound name/identifier |
| SMILES | str | Canonical SMILES notation |
| Targeted_Halogen | str | Halogen type (F, Cl, Br, or I) |

#### Target Variables
| Column | Type | Description |
|--------|------|-------------|
| BDE_kcal | float | Bond dissociation energy (kilocalories/mol) |
| BDE_kJ | float | Bond dissociation energy (kilojoules/mol) **PRIMARY TARGET** |

#### Molecular Descriptors (14 Features)
| Column | Type | Range | Description |
|--------|------|-------|-------------|
| MolWt | float | 50-500 | Molecular weight (g/mol) |
| LogP | float | -5 to 10 | Octanol-water partition coefficient (lipophilicity) |
| TPSA | float | 0-200 | Topological Polar Surface Area (Ų) |
| NumRotBonds | int | 0-30 | Number of rotatable bonds |
| HBD | int | 0-10 | Hydrogen bond donors count |
| FractionCsp3 | float | 0-1 | Fraction of sp3 hybridized carbons |
| NumHeavyAtoms | int | 5-200 | Count of non-hydrogen atoms |
| Halogen_Z | int | 9, 17, 35, 53 | Atomic number (F=9, Cl=17, Br=35, I=53) |
| C_degree | int | 1-4 | Connectivity degree of C bonded to halogen |
| C_hybridization | int | 2-4 | Hybridization of C (2=sp, 3=sp2, 4=sp3) |
| X_hybridization | int | 2-4 | Hybridization of halogen |
| C_GasteigerCharge | float | -1 to 1 | Computed partial charge on carbon |
| X_GasteigerCharge | float | -1 to 1 | Computed partial charge on halogen |
| X_EN | float | 2.6-4.0 | Electronegativity of halogen |

**Data Quality:**
- ✅ No missing values
- ✅ All descriptors are numeric
- ✅ Target variable (BDE_kJ) ranges from ~47 to ~135 kJ/mol
- ✅ Covers diverse C-X bond types and molecular environments

**Preprocessing:**
- Compounds standardized to canonical SMILES
- Descriptors computed using RDKit
- Halogen electronegativity from standard chemistry tables
- Gasteiger charges computed from 3D coordinate geometry

### External Validation Dataset (unseen_data.csv)

**Description:** Independent test set of molecules NOT seen during model training. Used to evaluate model generalization.

**Size:**
- Rows: ~50-100 molecules
- Columns: Same structure as full_dataset_14_descriptors.csv
- Memory: ~20-30 KB

**Use Case:** External validation to assess model performance on truly novel compounds.

**Data Quality:** Same preprocessing as training set.

### Intermediate Files

#### highly_correlated_pairs.csv

**Generated by:** `correlation_analysis.py`

**Content:** Pairs of descriptors with |Pearson r| > 0.9

**Columns:**
| Column | Example | Description |
|--------|---------|-------------|
| Descriptor_1 | MolWt | First descriptor in pair |
| Descriptor_2 | NumHeavyAtoms | Second descriptor in pair |
| Correlation | 0.95 | Pearson correlation coefficient |
| Abs_Correlation | 0.95 | Absolute value of correlation |

**Use:** Identifies redundant features for removal.

#### final_descriptor_list.csv

**Generated by:** `correlation_analysis.py`

**Content:** Final list of descriptors retained after correlation analysis

**Columns:**
| Column | Example | Description |
|--------|---------|-------------|
| Descriptor | MolWt | Descriptor name |
| Status | Retained | Status (always "Retained" in final list) |

**Use:** Specifies which 14 (or fewer) descriptors to use in model training.

## Feature Engineering Notes

### Computed Descriptors

The following descriptors are computed from molecular structure:
- **Gasteiger Charges** (C_GasteigerCharge, X_GasteigerCharge): Partial atomic charges computed using iterative Gasteiger-Marsili algorithm from RDKit
- **Hybridization states** (C_hybridization, X_hybridization): Derived from molecular connectivity and valence rules

### Standard Descriptors

These are standard computational chemistry descriptors:
- **MolWt, LogP, TPSA, NumRotBonds, HBD, FractionCsp3, NumHeavyAtoms** - RDKit Descriptors
- **Halogen_Z, X_EN** - From periodic table

## Preprocessing Steps

1. **Standardization:** SMILES canonicalization
2. **Descriptor Calculation:** RDKit + custom functions
3. **Imputation:** Mean imputation for any missing values (if any)
4. **Scaling:** MinMaxScaler applied during model preprocessing
5. **Feature Selection:** Removal of highly correlated descriptors

## Adding New Data

To add new molecules to the dataset:

1. **Prepare SMILES notation** - Canonical SMILES for each molecule
2. **Calculate descriptors** - Use RDKit/OpenBabel
3. **Add metadata** - Atoms, Targeted_Halogen columns
4. **Measure BDE** - Experimentally or computationally (DFT, etc.)
5. **Format** - Match CSV structure with all 19 columns

Example row:
```
,Tetrafluoromethane,FC(F)(F)F,F,127.5,105.5,1.3,0,0,0,1.0,10,4,4,3,0.25,-0.15,3.98
```

## Known Limitations

1. **Size:** Limited to ~400-500 training molecules (could benefit from larger dataset)
2. **Molecular diversity:** Primarily C-X bonds; limited to halogens (F, Cl, Br, I)
3. **Structural diversity:** Mostly organic compounds; may not extrapolate to highly exotic systems
4. **Computational cost:** TPOT training requires significant compute (~2-4 GB RAM)
